**Что тестирует:** Масштабируемость алгоритма (O(n³))

T(200) / T(100) ≈ 8
T(500) / T(200) ≈ 15.6

## 5. ТЕСТИРОВАНИЕ В ЛВС

### Подготовка
1. Создайте файл `workers.txt` с IP адресами:
```
192.168.1.10:11001
192.168.1.11:11002
192.168.1.12:11003
```

2. Откройте порты в брандмауэре:
```bash
netsh advfirewall firewall add rule name="SLAU Worker 11001" dir=in action=allow protocol=UDP localport=11001
netsh advfirewall firewall add rule name="SLAU Worker 11002" dir=in action=allow protocol=UDP localport=11002
netsh advfirewall firewall add rule name="SLAU Worker 11003" dir=in action=allow protocol=UDP localport=11003
```

### Запуск
1. На сервере: `ЗАПУСТИТЬ.bat`
2. На воркерах: `Worker_App\DistributedSLAU.Worker.exe 11001`

### Проверка
- Откройте http://localhost:5000
- Запустите воркеров
- Сгенерируйте систему 100×100
- Нажмите "Распределённое"

**Критерий:** Решение найдено, невязка < 10⁻⁸

## 6. ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Тесты не запускаются
**Ошибка:** `dotnet: command not found`
**Решение:** Установите .NET 8+ SDK

### Невязка слишком большая
**Причина:** Плохая обусловленность матрицы
**Решение:** Используйте диагонально доминантные матрицы

### Таймаут на больших размерах
**Причина:** Недостаточно памяти
**Решение:** Закройте другие приложения, увеличьте лимит памяти

### Воркеры не отвечают
**Причина:** Брандмауэр блокирует UDP
**Решение:** Откройте порты 11001-11003
